# x402 Payment Protocol

## Overview

x402 is a payment protocol that uses HTTP 402 status codes to challenge clients for payment before allowing access to resources.

## Flow

```
Client Request → Server
                ↓ (402 with x402-payment header)
    Challenge + Instructions
                ↓
    Client Signs Transaction
                ↓
    Payment Proof
                ↓ (200 with x402-proof header)
    Server Allows Access
```

## Challenge Response

```cpp
struct X402Challenge {
    std::string header;           // "x402-payment" or similar
    std::string challenge;         // Base64 encoded challenge
    std::string network;         // "mainnet", "devnet", etc.
    std::string currency;         // "USDC", "PYUSD", etc.
    uint64_t amount;            // Amount in minor units
    std::string recipient;         // Payment recipient
    std::optional<std::string> reference;   // Transaction reference
};
```

## Usage

```cpp
#include <dashpay/x402.hpp>

using namespace dashpay;

X402Client client("https://api.example.com");

auto challenge = client.parse_challenge(response_headers);
if (challenge) {
    auto signed = client.sign_challenge(*challenge, secret_key);
    auto proof = client.submit_payment(*challenge, *signed);

    // Retry request with x402-proof header
    request_headers.push_back({"x402-proof", proof.proof});
}
```

## Headers

| Header | Direction | Description |
|--------|-----------|-------------|
| x402-payment | Server → Client | Challenge from server |
| x402-proof | Client → Server | Payment proof from client |
| x402-accept | Server → Client | Supported payment types |
| x402-identity | Client → Server | Client identity for charge |

## Supported Networks

- Mainnet: `https://api.dash.org`
- Devnet: `https://api.devnet.dash.org`
- Testnet: `https://api.testnet.dash.org`
